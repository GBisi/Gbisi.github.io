Descrizione dettagliata

Rappresenta una sessione di un utente.

Una sessione definisce una singola connessione di un utente al gioco. Inizia con la creazione di una connessione al server da parte di un client. Gestisce tutte le richieste di un utente indirizzandole al servizio corretto o creando una sfida.

Una sessione si conclude con il logout.

◆ compute()

String Session.compute ( String  request)
private

Gestisce effettivamente la richiesta del client.

Controllare il protocollo di comunicazione nella relazione per maggiori dettagli.

Riguardo il protocollo di comunicazione il client invia un messaggio al server con il seguente formato: "nome_comando nome_utente [info_aggiuntive[...]] Dove le info aggiuntive possono essere anche una serie. Solitamente saranno il nome di un altro utente o la password dell'utente.

La risposta del server sara in caso di esito positivo: "OK [info_aggiuntive_richieste]" con le informazioni che sono opzionali. In caso di errore: "KO descrizione_errore" con descrizione non opzionale

requestLa richiesta del client

una stringa contenente la risposta del server alla richiesta o un messaggio di errore
